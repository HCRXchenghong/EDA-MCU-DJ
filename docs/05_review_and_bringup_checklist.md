# 审查与上电调试清单

## 1. 原理图审查清单

### 1.1 全局

- [ ] 所有 `TBD` 已关闭或明确不影响当前打样。
- [ ] 所有电源网命名一致。
- [ ] 所有页间连接有明确网名。
- [ ] 所有外部接口有连接器编号和针脚定义。
- [ ] 所有外部接口有 ESD/限流/滤波策略。
- [ ] 所有 MCU 电源脚、模拟电源脚、参考电压脚处理正确。
- [ ] 所有 MCU 有 SWD、NRST、BOOT 配置。
- [ ] 所有晶振负载、布线要求和备选时钟策略明确。

### 1.2 电源

- [ ] 电池输入极性、防反接、保险位置明确。
- [ ] 预充回路可控、可检测。
- [ ] 主接触器驱动有续流/TVS。
- [ ] 所有 DC/DC 输入耐压满足电池最高电压和浪涌余量。
- [ ] 24 V、15/12 V、5 V、3.3 V 电流预算完成。
- [ ] Gate Driver 电源上电顺序无明显问题。
- [ ] 外设 5 V/3.3 V 输出有限流。
- [ ] V_DOOR/V_LOCK 独立输入和板上供电切换逻辑明确。
- [ ] 母线电容额定电压和纹波电流校核。
- [ ] 刹车泄放/母线泄放预留。

### 1.3 急停与安全

- [ ] 急停输入不只进入 MCU。
- [ ] 急停能硬件拉低所有驱动 Enable。
- [ ] 急停能硬件断开主接触器控制链。
- [ ] 急停状态能被 H7 读取。
- [ ] Fault 锁存逻辑明确。
- [ ] 上电默认所有驱动不使能。
- [ ] MCU 复位时不会误打开 MOS/驱动。
- [ ] CAN 掉线后的硬件/固件停车策略明确。

### 1.4 轮毂/转向驱动

- [ ] 每个三相桥 Gate Driver 连接正确。
- [ ] 高低侧驱动、Bootstrap 或隔离供电方案正确。
- [ ] MOS Vds、Id、热设计有校核。
- [ ] 电流采样不是单母线主方案。
- [ ] 采样电阻功率和阻值合理。
- [ ] 采样放大器输入共模范围正确。
- [ ] 过流比较器能硬件关断。
- [ ] 母线电压采样分压耐压和功耗合理。
- [ ] Hall 输入有电平保护和滤波。
- [ ] 编码器接口有 ESD、串阻、供电滤波。
- [ ] 转向输出轴编码器是闭环主反馈。

### 1.5 通信

- [ ] CAN FD 收发器不是 TJA1050。
- [ ] CAN 终端电阻有跳帽。
- [ ] CANH/CANL 有 TVS。
- [ ] 外部 CAN 隔离电源方案完整。
- [ ] RS485 有终端和偏置预留。
- [ ] USB Type-C 的 D+/D-、CC1/CC2、VBUS 检测、ESD 和屏蔽接地完整。
- [ ] Ethernet PHY、RJ45/磁性器件、时钟、复位、ESD 和屏蔽接地完整。
- [ ] ESP32 天线净空要求写进 PCB 约束。
- [ ] UART 5 V 模式不会直连 MCU。

## 2. PCB 审查清单

### 2.1 功率布局

- [ ] 功率区靠板边。
- [ ] 每个三相桥本地母线电容靠近 MOS。
- [ ] 高 di/dt 回路面积最小。
- [ ] MOS 散热铜皮、过孔、散热片策略明确。
- [ ] 大电流路径宽度满足电流和温升。
- [ ] 功率地回流不穿模拟采样地。

### 2.2 采样与模拟

- [ ] 采样电阻 Kelvin 走线。
- [ ] 电流采样差分线短且远离开关节点。
- [ ] ADC 输入 RC 滤波靠近 MCU/放大器。
- [ ] 模拟参考地单点/分区策略明确。
- [ ] 母线电压采样远离开关节点。

### 2.3 通信与低压

- [ ] CAN/RS485 差分阻抗和走线一致性审查。
- [ ] USB Type-C 差分线、CC 走线、ESD 位置和连接器机械方向审查。
- [ ] Ethernet 差分线、PHY、RJ45/磁性器件布局合理。
- [ ] ESP32 天线禁铜和净空满足模块要求。
- [ ] UART/编码器/Hall 线不穿越功率区。

### 2.4 安规与制造

- [ ] 高压/大电流间距满足电池最高电压和污染等级目标。
- [ ] 连接器方向、防呆、丝印清晰。
- [ ] 测试点可接触。
- [ ] SWD 口可接触。
- [ ] 板边、安装孔、散热器避让正确。
- [ ] DRC 关键错误为 0。

## 3. 样板上电顺序

### 3.1 不接电机，不上主功率

1. 目检短路、方向、焊接。
2. 万用表测所有电源轨对地阻抗。
3. 限流电源给低压输入，验证 5 V/3.3 V。
4. 验证 H7 SWD 连接。
5. 验证 3 个 G4 SWD 连接。
6. 验证 ESP32 下载/启动。
7. 验证 USB Type-C 插入检测和 USB CDC 枚举。
8. 验证 Ethernet PHY 链路。

### 3.2 低压通信

1. H7 与前轮 G4 CAN 通信。
2. H7 与后轮 G4 CAN 通信。
3. H7 与转向 G4 CAN 通信。
4. H7 与 ESP32 UART 通信。
5. SPI-UART 扩展枚举。
6. RS485 收发。
7. 外部 CAN 收发。

### 3.3 安全链路

1. 急停输入读取。
2. 急停硬件拉低驱动 Enable。
3. 急停断开接触器控制。
4. Fault 锁存。
5. Fault 复位流程。
6. 上电默认禁用验证。

### 3.4 单路功率验证

每次只测一个电机通道：

1. 不接电机，只上 Gate Driver 电源，测 PWM/Gate。
2. 接假负载或小电机，限流启动。
3. 验证 Hall。
4. 验证电流采样零点和比例。
5. 验证过流关断。
6. Hall 六步低速运行。
7. 逐步增加电流。
8. 记录温升和波形。

### 3.5 FOC 验证

1. 单轮空载 FOC。
2. 单轮带载 FOC。
3. 双轮同区运行。
4. 前后轮四电机运行。
5. CAN 超时停车。
6. 电子刹车。
7. 长时间温升。

### 3.6 转向验证

1. 输出轴编码器读取。
2. 零位标定。
3. 限位开关验证。
4. 低电流位置控制。
5. 堵转保护。
6. 前后转向同步。

### 3.7 仓门与电子锁

1. 锁低压限流动作。
2. 锁状态检测。
3. 仓门限位读取。
4. 仓门电机低功率运行。
5. 开/关门超时保护。
6. 防夹输入验证。

## 4. 必须记录的数据

- 每条电源轨空载/满载电压。
- 每个 DCDC 温升。
- 每个 MOS 温升。
- 每个采样通道零点和比例。
- CAN 错误计数。
- 急停响应时间。
- Fault 触发和释放逻辑。
- 电机启动电流、峰值电流、稳态电流。
- 布线/EMI 相关异常波形。

## 5. 二版改板来源

二版改板项应只来自：

- 样板上电记录。
- 台架测试记录。
- 设计审查未关闭项。
- 器件不可采购或不可制造问题。
- 用户需求变更。
