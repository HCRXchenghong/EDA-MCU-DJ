# 决策记录

## 使用方法

每个会影响原理图、PCB、BOM、固件或安全策略的选择都记录在这里。

字段说明：

- `状态`：Proposed、Accepted、Rejected、Superseded。
- `依据`：来自用户输入、数据手册、测试、审查，或工程约束。
- `影响`：会影响哪些文档、原理图页、PCB 区域或固件模块。

## D001：采用一块大 PCB，内部模块化分区

| 字段 | 内容 |
|---|---|
| 状态 | Accepted |
| 决策 | 底盘控制采用一块大 PCB，不拆成多个独立小板 |
| 依据 | 原始需求明确 |
| 影响 | PCB 需要严格分区；功率区、低压主控区、无线区、电源安全区要隔离 |
| 风险 | 板子面积大，EMI 和热设计复杂，返修成本高 |
| 后续动作 | PCB 前必须冻结板框、连接器位置、散热结构 |

## D002：STM32H7 作为唯一主控

| 字段 | 内容 |
|---|---|
| 状态 | Accepted |
| 决策 | 上位机只与 STM32H7 通信，H7 负责调度、安全、仓门、电锁 |
| 依据 | 原始需求明确 |
| 影响 | H7 需要 Ethernet、USB Type-C、FDCAN、SPI、UART、PWM/ADC、GPIO 大量资源 |
| 风险 | H7 资源可能紧张，尤其加上仓门电机控制后 |
| 后续动作 | 选高引脚数 H7，并做完整 PinMux 表 |

## D003：ESP32-S3 只做无线通信从模块

| 字段 | 内容 |
|---|---|
| 状态 | Accepted |
| 决策 | ESP32-S3 不直接控制任何执行器 |
| 依据 | 原始需求明确 |
| 影响 | ESP32 只接 H7，不进入安全链路 |
| 风险 | 无线调试功能受 H7 协议限制 |
| 后续动作 | 定义 H7-ESP32 通信协议 |

## D004：轮毂/转向驱动分配给 3 个 STM32G4

| 字段 | 内容 |
|---|---|
| 状态 | Accepted with Risk |
| 决策 | 前轮、后轮、转向分别由一个 STM32G4 双路控制 |
| 依据 | 原始需求明确 |
| 影响 | 每个 G4 必须支持双路 FOC 资源 |
| 风险 | 单 G4 双 FOC 对高级定时器、ADC、DMA、COMP、PinMux 要求高 |
| 后续动作 | 具体 G4 型号不能猜，必须按资源预算选型 |

## D005：FOC 电流采样不采用单母线主方案

| 字段 | 内容 |
|---|---|
| 状态 | Accepted |
| 决策 | 首版硬件按三低边或三相采样设计 |
| 依据 | 原始需求明确提出不建议只做单母线采样 |
| 影响 | 每个电机通道需要更多采样电阻、放大器、ADC 资源 |
| 风险 | PCB 面积和布线复杂度增加 |
| 后续动作 | 峰值电流确认后计算采样电阻和放大器增益 |

## D006：急停必须硬件关断

| 字段 | 内容 |
|---|---|
| 状态 | Accepted |
| 决策 | 急停不能只进 MCU，必须硬件关断驱动 Enable 和主接触器 |
| 依据 | 原始需求明确 |
| 影响 | 需要独立 Safety Chain 原理图页 |
| 风险 | 若锁存/复位策略不清晰，会导致误恢复或无法恢复 |
| 后续动作 | 等用户确认急停解除后的恢复策略 |

## D007：CAN FD 不使用 TJA1050

| 字段 | 内容 |
|---|---|
| 状态 | Accepted |
| 决策 | 全板 CAN 按 CAN FD 设计，不用 TJA1050 |
| 依据 | 原始需求明确 |
| 影响 | 内部/外部 CAN 收发器按 CAN FD 选型 |
| 风险 | 外部 CAN 隔离和终端策略需确认 |
| 后续动作 | 确认 CAN FD 速率、线长、拓扑 |

## D008：转向角闭环必须测输出轴

| 字段 | 内容 |
|---|---|
| 状态 | Accepted |
| 决策 | 转向闭环主反馈为输出轴绝对编码器，不只测电机轴 |
| 依据 | 原始需求明确 |
| 影响 | 每个转向输出轴必须有绝对编码器接口和机械安装方案 |
| 风险 | 机械偏心、磁铁安装、防水防松会影响控制精度 |
| 后续动作 | 等用户确认编码器型号和机械结构 |

## D009：仓门电机由 H7 直控

| 字段 | 内容 |
|---|---|
| 状态 | Accepted with Risk |
| 决策 | 仓门 5-24 V 无刷编码电机由 H7 直接控制 |
| 依据 | 原始需求明确 |
| 影响 | H7 需要额外 PWM、ADC、保护和驱动逻辑 |
| 风险 | H7 资源紧张；仓门功率噪声可能影响主控 |
| 后续动作 | 确认仓门电机是否必须 FOC；若资源不足，需重新评估独立小 MCU |

## D010：外部 UART 支持 3.3 V/5 V 跳线但 MCU 侧固定 3.3 V

| 字段 | 内容 |
|---|---|
| 状态 | Accepted |
| 决策 | 外部 UART VOUT 和信号电平随跳线选择，MCU/桥接芯片侧固定 3.3 V |
| 依据 | 原始需求明确 |
| 影响 | 每路 TX/RX 必须电平转换 |
| 风险 | 自动方向电平转换器在 UART 上可能带来边沿/驱动问题，需选合适器件 |
| 后续动作 | 根据波特率和方向选电平转换器 |

## D011：必须提供 Ethernet 网口

| 字段 | 内容 |
|---|---|
| 状态 | Accepted |
| 决策 | 底盘控制大板必须保留 1 个 Ethernet 网口 |
| 依据 | 用户明确要求“有没有网口？没有的话给我加上” |
| 影响 | H7 型号/封装必须支持 Ethernet MAC；通信页必须包含 PHY、RJ45/磁性器件、ESD、时钟和复位 |
| 风险 | Ethernet 会增加 H7 引脚压力、PCB 差分布线和 EMC 压力 |
| 后续动作 | 选定 PHY、RMII/MII、RJ45 是否带磁、是否带灯以及屏蔽接地策略 |

## D012：必须提供 USB Type-C

| 字段 | 内容 |
|---|---|
| 状态 | Accepted |
| 决策 | 底盘控制大板必须提供 1 个 USB Type-C 接口 |
| 依据 | 用户明确要求“还有 typec” |
| 影响 | 通信页必须包含 Type-C 连接器、CC 下拉、VBUS 检测、USB ESD 和屏蔽接地 |
| 风险 | 若后续要求 USB HS/供电/双角色，会增加器件和布线复杂度 |
| 后续动作 | 首版按 USB 2.0 FS Device/CDC 规划，后续确认是否需要 HS 或 Type-C 供电能力 |
